‚öΩ Troubleshooting

A. In Deployment :

1. Mixed content block: I initially hosted the deployment domain with HTTPS, but since the server address protocol was HTTP, there was a mixed content block due to the mismatch in protocols between the server and the client. To resolve this, I deployed the server with HTTPS as well.
Although I was initially confused by the mixed content block error, I realized the importance of protocols in internet communication and how they can affect functionality.

2. CORS: Since I deployed the server application on an EC2 instance, I configured allowed origins in Spring Security to allow access from specific origins. I also configured CORS for S3 bucket on AWS. However, even after these configurations, CORS errors persisted when accessing the deployment link. After extensive research, I found a clue on Stack Overflow that suggested adding policies for CloudFront when deploying S3 website hosting via CloudFront. Finally, by adding a policy for CloudFront to my bucket policy, I resolved the issue.
This experience made me realize that when using AWS services, there are various considerations to ensure proper configuration and functionality. It also highlighted that CloudFront provides more than just hosting capabilities.

3. AWS RDS external connection block: After completing deployment, I encountered issues with external connections to RDS from my local environment without any apparent reason or error message. Despite allowing all inbound rules for RDS during initial deployment setup, external connections were not established. Unable to find a solution through search alone, while reading through AWS official documentation step by step, I came across a statement suggesting that incorrect subnet settings could cause failures in external connections to RDS. As a result, I created new subnets and routing tables directly in AWS Virtual Private Cloud (VPC) and associated them with my RDS instance. Afterward, successful connections were established from my local environment.
This experience helped me understand subnet and routing table relationships within computer networks and emphasized their importance for developers' understanding of networking concepts.

4. AWS account suspension: During the process of configuring deployment using AWS services, my account was suspended twice.
The first suspension occurred when accidentally including an AWS access key in my local code while connecting to AWS services locally. Although aware of not exposing sensitive environment variables like access keys publicly, it turned out that an access key was present in one of my log files unbeknownst to me until after pushing it to GitHub repository which resulted in account suspension. Immediately upon discovering this issue through email notification from AWS, I removed both exposed access key files as well as deleting exposed keys themselves and contacted AWS support requesting reinstatement after resolving this issue. Through these experiences , i realized importance maintaining secrecy server environment variables furthermore , all remaining variables were hidden locally using parameter store provided by aws . Additionally , i became aware challenges managing multiple variables when utilizing external services as well recognized significance devops professionals their efforts field.

Although swap area enabled instances , traffic continued overwhelm them resulting unresponsiveness . üìö Summary

While creating my portfolio, I felt that using PDF or other static representations was not enough, so I decided to create a portfolio website from scratch.
Since I had no prior experience in front-end development, I had to study HTML, CSS, and JavaScript to some extent and used Bootstrap templates for the design.
Although the implementation was done without HTTP communication, there were many aspects that required attention, making it challenging.
On September 19, 2023, I implemented automatic deployment through Github Actions.

üìë Meaning

This project was my first attempt at using technologies that I had no previous experience with.
Initially, I planned to develop the website using the React framework. However, after considering that a simple static hosting would suffice without requiring HTTP communication, I opted for static hosting. Since my knowledge of design was limited at the time, I used Bootstrap templates for the layout.
Configuring CSS and specifying styles for each component was confusing and required careful attention. However, seeing the completed website gave me a sense of accomplishment, and it also provided me with a better understanding of the challenges faced by front-end developers.
For deployment, I used AWS S3 bucket for static website hosting. I purchased a domain and obtained an SSL certificate from ACM to deploy via CloudFront with HTTPS. Finally, I hosted the S3 endpoint on Route 53 using my purchased domain.
In conclusion, as an additional thought about this project, I am considering refactoring it into a web application where authenticated users can write blog-style articles by implementing user authentication functionality in the future.
